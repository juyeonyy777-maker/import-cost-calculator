# CN 수입 원가 계산기

## 프로젝트 개요
CNINSIDER 중국 수입 원가를 자동 계산하는 웹 프로그램 (Next.js)

## 실행 방법
- 바탕화면 > `클로드 전용 260320` > `수입원가 계산기.bat` 더블클릭
- 또는 이 폴더에서 `npx next dev -p 3001 -H 0.0.0.0`
- 내 컴퓨터: http://localhost:3001
- 같은 와이파이: IP가 바뀔 수 있으므로 `ipconfig`로 확인 후 `http://{IP}:3001`
- 방화벽 규칙 `ImportCostCalculator3001`, `NodeJSAllow` 등록됨

## 주요 파일
- `app/page.js` — 메인 화면 (이름 입력, 파일 업로드, 계산, 결과 테이블)
- `app/api/calculate/route.js` — 계산 API (파일 형식 검증, 엑셀 파싱, 위안화 비교, 원가 계산, 평균 원가, 최근5건 비율)
- `app/api/save-all/route.js` — 전체 데이터 저장/조회 API
- `app/api/save-all/download/route.js` — 전체 데이터 엑셀 다운로드
- `app/api/log/route.js` — 사용 로그 API
- `app/data/page.js` — 전체 데이터 조회 페이지 (검색 + 전체 헤더 정렬 + 평균 원가)
- `app/logs/page.js` — 사용 로그 조회 페이지
- `app/guide/page.js` — 사용방법 페이지
- `app/logic/page.js` — 컬럼별 계산 로직 정리
- `lib/calculator.js` — 수입원가 계산 엔진 (상품별 환율 적용)
- `lib/parsers/excel-parser.js` — 결제명세서/출고내역 엑셀 파서 (환율 Y열 추출)
- `lib/parsers/invoice-parser.js` — 청구서 PDF 파서
- `lib/parsers/declaration-parser.js` — 수입정산서 PDF 파서

## 현재 기능
1. **이름 입력** — 첫 접속 시 이름 입력, localStorage 저장, 오른쪽 상단에 표시
2. **파일 업로드** — 통합 드래그앤드롭 (결제명세서/출고내역/청구서 필수, 수입정산서 제거됨), ZIP 자동 해제, 파일명으로 자동 분류
3. **파일 형식 검증** — 결제명세서(SKU/단가/총금액/환율), 출고내역(SKU/출고수량), 청구서(KRW/TOTAL) 필수 항목 확인, 안 맞으면 에러 팝업 + 작동 안 함
4. **환율** — 결제명세서 Y열(24번 인덱스)에서 상품별 환율 자동 적용 (환율 입력란 없음)
5. **원가 계산** — SKU별 수입원가 자동 계산
6. **위안화 비교** — 단가(CNY)와 `위안화 정보.xlsx` I열 비교, 4위안 이상 차이시 빨간색 + 괄호로 엑셀 값 표시
7. **비용 요약** — 최근5건 평균비율(건별 상세 포함), 출고코드(AE260310-331박스), 총수입원가, 총제품비(CNY=총금액+운임, 원화환산), 총부대비용, 총수량, 전체CBM
8. **총 수입원가** = 각 SKU의 원가(개당) × 수량 합계 (배분총합과 동일)
9. **총 제품비(CNY)** = 결제명세서 총금액 합 + 운임 합 (출고내역 CNY와 동일)
10. **오차 표시** — 배분합 vs 청구서, 계산기 vs 출고내역 CNY 비교 (+ 파란색, - 빨간색)
11. **테이블 컬럼** — SKU, 개당CBM, 품명(정렬), 수량, 단가(CNY), 후불작업비용, 수수료7%, 원가(개당), 평균 원가, 위안화 비율, 원가(x285), 수수료1%~내륙운송료
12. **평균 원가** — 저장된 전체 데이터에서 해당 SKU의 원가(개당) 평균
13. **위안화 비율** — 원가(개당) ÷ 단가(CNY), 실제비용 행에 `평균` 표시
14. **최근5건 평균비율** — 출고코드 날짜순 최근 5건의 평균 위안화 비율 + 건별 상세
15. **EXCEL 다운로드** — 맨 앞 열에 출고코드 포함
16. **자동 저장** — 계산 성공 시 자동 저장 (박스수량 불일치 등 에러시 저장 안 함, 같은 출고코드 덮어쓰기)
17. **전체 데이터 조회** — 저장된 모든 데이터 바로 표시 + 검색 + 모든 헤더 정렬(오름차순/내림차순) + 평균 원가 + 엑셀 다운
18. **사용 로그** — 로그인, 파일업로드, 계산, 자동저장, 다운로드, 초기화 기록, /logs에서 조회
19. **사용방법** — /guide 페이지, 첫 화면에 버튼

## 계산 로직 요약
- **단가(CNY)** = 결제명세서 총금액 ÷ 세트수량 + 운임단가
- **원가(개당)** = (단가CNY + 0.7 후불작업비용 + 수수료7%) × 상품별환율 + 배분비용(개당)
- **총 제품비(원화)** = 각 SKU의 (단가CNY + 0.7 + 수수료7%) × 상품별환율 × 수량 합
- **총 수입원가** = 각 SKU의 원가(개당) × 수량 합계
- **위안화 비율** = 원가(개당) ÷ 단가(CNY)
- 비용 배분: 해상운임/내륙운송 = CBM 비례, 나머지 = 수량 N분의 1, 부가세 = 과세가격 비율
- 환율: 결제명세서 Y열(인덱스 24)에서 상품별 적용

## 개당 CBM 계산
- 출고내역에서 상자별 CBM을 상자 안 총 수량으로 나눔
- 같은 상자 안 상품은 동일한 개당 CBM
- CBM 0인 행도 같은 상자의 CBM을 공유받음 (상자 첫 행의 CBM 사용)

## 위안화 비교 로직
- `위안화 정보.xlsx`에서 SKU(F열 바코드) → 위안화(I열) 매핑
- API에서 `fs.readFileSync` + `XLSX.read(buffer)` 방식으로 읽음 (readFile은 Next.js에서 안 됨)
- 차이 4위안 이상이면 단가 빨간색 + 괄호로 엑셀 값 표시
